# 2026-04-02 — 블로그 검색 기능 추가

## 목표

블로그(지식 모음)와 스토리(나의 이야기) 두 콘텐츠 타입의 MDX 포스트를 메타데이터 기반으로 검색하는 기능 추가. 헤더에 돋보기 아이콘 → `/search` 페이지 이동 → 실시간 필터링 결과 표시.

## 구현 단계

### 1단계: 타입 및 데이터 레이어

- `features/blog/types.ts`에 `SearchablePost` 인터페이스 추가 (`PostMeta` + `href`)
- `features/blog/lib/posts.ts`에 `getAllPostsForSearch()` 함수 추가
  - 블로그(`/content/blog/`) + 스토리(`/content/stories/`) 전체 합산
  - 각 포스트에 올바른 `href` 부여 (`/blog/{slug}`, `/stories/{slug}`)

### 2단계: 기존 컴포넌트 확장

- `BlogCard`에 `href?: string` 옵션 prop 추가 (없으면 기존 `/blog/${slug}` 유지)
- 기존 사용처(BlogList 등)에 영향 없음

### 3단계: 헤더 수정

- `SiteHeader.tsx`에 돋보기 아이콘(`HiOutlineMagnifyingGlass`) 추가
- 테마 토글 왼쪽에 배치, 기존 `themeToggle` 스타일 재사용
- `aria-label="검색 페이지로 이동"` 접근성 처리

### 4단계: 검색 페이지 생성

- `app/search/page.tsx` — 서버 컴포넌트, `getAllPostsForSearch()` 호출 후 props 전달
- `app/search/search.css.ts` — 기존 `blog.css.ts` 패턴 재사용 (단일 컬럼)
- `features/search/components/SearchContent.tsx` — 클라이언트 컴포넌트
  - `useState`로 검색어 관리, 입력할 때마다 실시간 필터링
  - `BlogCard`에 `href` prop 전달하여 올바른 상세 페이지 링크
- `features/search/components/SearchContent.css.ts` — 검색 input, 결과 리스트 스타일

## 검색 로직

```
filterAndSort(posts, query):
  q = query.toLowerCase()
  각 post에 대해:
    titleMatch = post.title.toLowerCase().includes(q)
    descMatch = post.description.toLowerCase().includes(q)
    tagMatch = post.tags.some(tag => tag.toLowerCase().includes(q))

    매칭 없으면 제외
    priority = titleMatch ? 1 : descMatch ? 2 : 3

  priority 오름차순 정렬 (낮을수록 상위)
```

- 대소문자 무시: 모든 비교를 `.toLowerCase()`로 통일
- 중복 방지: 같은 포스트가 여러 필드에 매칭돼도 한 번만 표시 (최고 우선순위 적용)

## 주의사항

- 스타일은 반드시 Vanilla Extract (`*.css.ts`)로 작성, 인라인 스타일 지양
- 서버/클라이언트 컴포넌트 분리: 페이지는 서버 컴포넌트, 인터랙션은 클라이언트 컴포넌트
- `BlogCard`의 `href` prop은 옵션이므로 기존 코드에 breaking change 없음
- 검증: `npm run lint` → `npm run format:check` → `npm run build` 순서로 확인
