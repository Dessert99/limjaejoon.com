# 2026-04-03 — SEO 최적화

> **목표:** 블로그가 구글 검색에 잘 노출되도록 메타데이터, sitemap, robots.txt, 구조화 데이터를 한번에 추가하여 검색 엔진 최적화를 완성한다.

---

## Context
현재 블로그에 SEO 설정이 전혀 없어서 구글 검색에 제대로 노출되지 않는 상태. 메타데이터, sitemap, robots.txt, 구조화 데이터를 한번에 추가하여 검색 엔진 최적화를 완성한다.

## Step 0: 사이트 URL 상수 정의
**새 파일:** `features/shared/constants.ts`
```ts
export const SITE_URL = 'https://limjaejoon.com';
```
- sitemap, robots, metadata, JSON-LD에서 공통으로 사용

## Step 1: 루트 메타데이터 — `app/layout.tsx`
`metadata` export 추가:
- `metadataBase`: `new URL(SITE_URL)` — 상대 경로 OG 이미지가 절대 URL로 변환됨
- `title.default`: `'임재준 | 프론트엔드 개발자'`
- `title.template`: `'%s | 임재준'` — 하위 페이지에서 title만 설정하면 자동 조합
- `description`: `'프론트엔드 개발자 임재준의 기술 블로그입니다.'`
- `openGraph`: type, locale(ko_KR), siteName, 기본 이미지(`/images/logo.png`)
- `twitter`: card summary

## Step 2: 페이지별 정적 메타데이터
| 파일 | title | description | 비고 |
|------|-------|-------------|------|
| `app/page.tsx` | (layout default 상속) | 별도 설명 추가 가능 | 변경 최소화 |
| `app/blog/page.tsx` | `'지식 모음'` | `'개념 정리와 레퍼런스를 모아두는 공간입니다.'` | |
| `app/portfolio/page.tsx` | `'포트폴리오'` | `'프론트엔드 개발자 임재준의 기술 스택과 활동 이력입니다.'` | |
| `app/search/page.tsx` | `'검색'` | — | `robots: { index: false, follow: false }` |

## Step 3: 동적 메타데이터 — `[slug]` 페이지
**`app/blog/[slug]/page.tsx`** 및 **`app/stories/[slug]/page.tsx`**에 `generateMetadata` 추가:
- `title`: post.title (template에 의해 "제목 | 임재준" 으로 렌더링)
- `description`: post.description
- `keywords`: post.tags
- `openGraph`: type `article`, publishedTime, tags, url
- `twitter`: card summary

## Step 4: robots.txt — `app/robots.ts` (새 파일)
```ts
rules: { userAgent: '*', allow: '/', disallow: '/search' }
sitemap: `${SITE_URL}/sitemap.xml`
```

## Step 5: sitemap.xml — `app/sitemap.ts` (새 파일)
- 정적 페이지: `/` (priority 1.0), `/blog` (0.8), `/portfolio` (0.5)
- 블로그 포스트: `getPostList()` → priority 0.7
- 스토리 포스트: `getStoryList()` → priority 0.6
- `/search`는 제외

## Step 6: JSON-LD 구조화 데이터
**`app/blog/[slug]/page.tsx`** 및 **`app/stories/[slug]/page.tsx`**의 JSX에 `<script type="application/ld+json">` 추가:
```json
{ "@type": "Article", "headline", "description", "datePublished", "author": { "@type": "Person", "name": "임재준" }, "url", "keywords" }
```

## 검증
1. `npm run lint` — 에러 없음 확인
2. `npm run build` — 빌드 성공 확인
3. 개발 서버에서 확인:
   - 홈페이지 `<title>` → "임재준 | 프론트엔드 개발자"
   - 블로그 포스트 `<title>` → "{포스트 제목} | 임재준"
   - 블로그 포스트 `<head>`에 og:type=article, JSON-LD script 존재
   - 검색 페이지 `<meta name="robots" content="noindex, nofollow">`
   - `/robots.txt` 정상 응답
   - `/sitemap.xml` 모든 페이지 포함 확인
