# 학습 정리 노트 생성

현재 대화에서 수행한 작업을 학습 정리 문서로 만드는 스킬.

- **대상 독자**: 작성자 본인 (미래의 나)
- **템플릿**: 이 스킬 폴더의 `guideline.md` (반드시 읽어서 형식 규칙을 따른다)
- **출력 위치**: `docs/reports/YYYY-MM-DD-{주제 영문 kebab-case}.md`
- **컨텍스트**: 현재 대화에서 수행한 작업 내용 전체

## 실행 흐름

### 1. guideline.md 읽기

이 스킬 폴더의 `guideline.md`를 읽어 문서 작성 규칙을 확인한다.

### 2. 주제 확인

- `/study-note [주제]`로 주제가 주어지면 그것을 사용한다.
- 인자가 없으면 현재 대화 내용에서 주제를 추론하고 사용자에게 확인한다.

### 3. 소제목 구성 제안

현재 대화의 작업 내용이 곧 컨텍스트이므로, 바로 소제목(## 단위) 구성을 제안한다.
사용자가 추가하거나 빼고 싶은 내용이 있는지 확인한다.

### 4. 섹션별 작성

guideline.md 형식 규칙에 따라 섹션별로 작성한다:

- 소제목은 `##` 형태, 소제목마다 번호를 1번부터 새로 시작
- 한 문장씩 끊어서 번호 (1. 2. 3. …)
- 각 문장은 앞 문장과 유기적으로 연결
- 코드 블록은 원리 이해에 필요한 로직만 삽입
- 마지막에 `→ (핵심 요약)` 추가

각 소제목마다 guideline.md의 내용 구성 규칙 중 적합한 유형을 적용한다:
- ① 새로운 개념 → 배경 → 문제 → 개념 소개 → 정의, 작동 방식
- ② 문제 해결 → 문제 → 원인 → 해결
- ③ 비교 → 공유 맥락 → 차이점
- ⑤ 복습 → 핵심 정의와 작동 방식만

섹션별로 사용자 피드백을 받으며 수정한다.

### 5. 최종 검수

전체 문서를 다시 읽고 아래를 확인한다:
- 흐름과 일관성
- 중복이나 모순
- 모든 문장이 가치를 갖는지

### 6. 파일 저장

`docs/reports/YYYY-MM-DD-{주제}.md`로 저장한다.
같은 날짜에 같은 주제가 있으면 접미사를 붙인다.

## 주의사항

- "~했다" 서술보다 "왜 그렇게 했는지"에 초점
- 사용자가 처음 보는 개념은 배경부터 설명
- 대화에서 실제로 다룬 내용만 포함, 추측 금지
