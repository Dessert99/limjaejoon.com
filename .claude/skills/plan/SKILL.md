# /plan — 기능 단위 설계 사이클

이 스킬은 **설계 단계만** 담당한다. 코드는 작성하지 않는다.
실제 구현은 `/execute`가 state.md를 읽고 이어간다.

## 입력
`/plan <기능명 한 줄>` — 예: `/plan 로그인-인증-세션 로직`
인자가 없으면 사용자에게 묻는다.

## 출력
`docs/plans/<번호>-<슬러그>/`
- `PRD.md` — 기능 요구사항
- `adr/0001-*.md`, `0002-*.md`, ... — 의사결정들
- `state.md` — 다음 세션 핸드오프

## 실행 흐름

### 1. plan 폴더 생성

1. `docs/plans/` 폴더 스캔(`_` 시작 폴더 제외) → 다음 번호 결정 (`0001`, `0002`, ...).
2. 사용자에게 슬러그 제안 (kebab-case, 영문). 예: "로그인-인증-세션" → `login-auth-session`. 확인 받기.
3. `docs/plans/<번호>-<슬러그>/` 폴더 생성.

### 2. PRD 작성

1. `.claude/skills/plan/templates/PRD.md`를 복사해 `docs/plans/<plan>/PRD.md`로 저장.
2. **사용자와 대화하며** 채운다:
   - 목표: 한 줄로
   - **범위**: frontend / backend / infra 중 어디를 건드리는지 체크 — 자동 라우팅의 핵심 신호
   - **보안 민감도**: 5개 항목 중 해당되는 것 체크 — security-auditor 호출 트리거
   - 완료 기준: 검증 가능한 항목으로
   - 비범위: 이번에 안 할 것
3. 모호하면 추측하지 말고 묻는다 (CLAUDE.md 원칙).

### 3. 자동 라우팅 결정

PRD를 읽고 호출할 에이전트를 결정한다. 결과는 4단계 ADR 작성 시 도메인 에이전트 호출, 5단계 검토 시 보안 검토자 호출에 쓴다. state.md "자동 라우팅 결과" 섹션에도 기록.

**도메인 에이전트** (범위 체크박스 기반):
| PRD scope | 호출 에이전트 |
|---|---|
| frontend ✓ | `nextjs-developer` |
| backend ✓ | `node-specialist` |
| infra ✓ | `devops-engineer` |

**보안 검토자** (보안 민감도 체크박스 기반):
보안 민감도에 하나라도 체크가 있으면 → `security-auditor` 호출 (설계 단계).

**접근성 검토자**:
plan 단계에서는 호출하지 않음. execute 단계에서 frontend가 포함된 경우만 호출.

### 4. 도메인 에이전트로 설계 의견 수집

3단계에서 결정된 도메인 에이전트들을 **병렬로 호출**한다. 각 에이전트에 다음을 전달:
- PRD.md 전체
- 해당 도메인이 다뤄야 할 부분만 명시 ("frontend 부분만 설계 의견 줘")
- 출력 형식: 옵션 비교 + 추천안 + 트레이드오프

여러 에이전트가 충돌하는 결정을 내면 (예: API 응답 스키마) 사용자에게 보여주고 조정.

### 5. ADR 초안 작성

도메인 에이전트 의견을 바탕으로 결정 사항을 ADR로 정리.
- 한 ADR = 하나의 의사결정
- 파일명: `adr/0001-<slug>.md`, `0002-...` (plan 폴더 안에서 1번부터)
- 템플릿: `.claude/skills/plan/templates/adr.md` 복사
- 사후 회고는 study-note(`reports/`)에서 일원화 — ADR에 별도 회고 섹션 두지 않음
- 사용자 피드백 받으며 수정

### 6. 검토 라운드 (1회)

- **항상**: `architect-reviewer` 호출 — 모듈 경계, 의존성 방향, ADR 정합성 검토
- **조건부**: 보안 민감도 체크가 있으면 `security-auditor` 호출 — 인증 흐름, 권한 모델, 데이터 노출 표면 검토 (설계 관점)

병렬 호출. 결과를 사용자에게 보여주고:
- 채택할 피드백 → ADR 수정
- 무시할 피드백 → 사유 한 줄 메모

**재검토 라운드 상한: 1회.** 추가 라운드는 사용자 명시적 요청 시만.

### 7. state.md 작성

`.claude/skills/plan/templates/state.md` 복사 → 채움:
- 기능명, 관련 문서 경로
- 현재 단계: PRD ✅, ADR 초안 ✅, ADR 검토 ✅
- 자동 라우팅 결과 (3단계에서 결정한 것)
- 마지막 작업 요약: "ADR 검토 완료, 구현 대기"
- 다음 액션: `/execute`가 즉시 시작할 수 있는 단위로 명령형 작성
- 건드린 파일: PRD/ADR 경로
- 미해결 결정/질문: 보류된 항목

### 8. 보고

사용자에게 요약 출력:
- 생성된 plan 폴더 경로
- 작성된 ADR 목록
- 다음 단계: 새 세션에서 `/execute <plan-id>` 호출

## 주의사항

- **코드 작성 금지**. 설계만.
- **추측 금지**. 모호하면 묻는다.
- ADR과 도메인 에이전트 의견이 충돌하면 사용자가 결정.
